Open Hardware & Firmware: Steuerung im Browser, Automatisierung per MQTT, Firmware-Updates über WLAN — der Access-Point bleibt für die Feldkonfiguration erhalten.

Quellcode df3mt.de

Alles aus einem Projekt

Mechanik, Elektronik und Software sind aufeinander abgestimmt — vom Aufbau bis zum Field Day.

Responsive Web-Oberfläche

Touch-optimierte Bedienung, PWM-Rampen und eine Sitzung pro Browser: eine Person hält die Steuerung, ohne die Oberfläche für andere dauerhaft zu sperren.

SoftAP + Heim-WLAN

Verbindung zum Router, während der Konfigurations-AP aktiv bleibt. Der DHCP-Hostname erscheint in der Geräteliste.

MQTT & Home Assistant

Automatische Geräteerkennung, Befehle auf /set, Rückmeldung auf /state. Ungültige Nutzdaten werden verworfen.

Updates über die Luft (OTA)

.bin-Upload im Browser oder Arduino-IDE-Netzwerkport (mDNS). Optionales Passwort im Flash (NVS).

3D-druckbare Mechanik

CAD- und Druckdaten im Ordner 3D Model — anpassbar an Mast und Last.

Robust im Betrieb

Drosselung störender Clients, yield() beim langen Web-Flash, serielles Log für die Fehlersuche.

Architektur im Überblick

Typischer Datenfluss vom Bediener bis zur H-Brücke (z. B. L298N-Klasse).

Erste Schritte

Kurzer Weg von den Bauteilen zur ersten Bewegung.

  1. Mechanik aus 3D Model/ drucken und montieren.
  2. ESP32 mit Motortreiber verdrahten; GPIOs in der Firmware setzen.
  3. Mit Arduino IDE flashen (ESP32-Core 3.x, OTA-Partitionsschema).
  4. Mit dem Geräte-AP verbinden, Captive Portal / Statusseite öffnen, optional Heim-WLAN eintragen.
  5. MQTT in der UI aktivieren, falls Home Assistant genutzt wird.

Ausführliche Hinweise stehen in der README im Ordner Firmware/ auf GitHub.
